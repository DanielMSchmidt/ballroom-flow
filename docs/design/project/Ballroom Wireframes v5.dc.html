System rules — how everything behaves
ATTRIBUTE MODEL
A moment (a count, incl. off-beats & / a) holds typed attributes: Step (direction + footwork, no foot side), Rise&Fall, Position, Foot Position (ballet 1st–5th), Body actions, Sway, Turn — plus user custom types. Each type is single/multi, optional or required, and may be role-aware. Three value behaviours: overview shows a tight code (HT, PP, ¼R); editing shows the full label (Heel-Toe); tapping a value — or a column header — opens its explanation (1.13). Size = bars.
DANCES
Each choreo is one dance, chosen at creation. Phrase length: Waltz & Viennese = 3 or 6 bars; Quickstep, Foxtrot, Tango = 4 or 8. You only see figures valid for that dance. Many figures share a name across dances (Natural Turn, Fallaway) but carry different steps per dance.
FIGURE COPIES — global › choreo-owned › personal library
global = app-owned canonical library figure; nobody edits it. choreo-owned copy = the home of any edited / custom figure — lives in ONE routine, edited in place by everyone on that choreo, never ripples out. personal library = saved by a deliberate "↟ Save to My figures", reused across your choreos. Copy-on-write by location: editing a figure that lives outside this choreo auto-makes a frozen choreo-owned copy (toast "Copied into this choreo"); the source is untouched.
ROUTINES — share vs fork
Share: partners see each other's notes live; editing a shared figure changes it for everyone sharing this routine. Fork ("make it your own"): a frozen, independent copy you fully own — header shows "forked from <origin> · independent copy"; changes never flow back.
NOTES & LINKING
v1: a note attaches to a place (step 2 of this Natural Turn — one spot, no scope) or a figure, with a scope toggle: this choreo · all <dance> · every dance (one path covering a one-routine note and a cross-dance figure-family note). Later (v1.1): linking to an attribute value across steps ("all CBMPs", "all left sways") — needs a matching engine, deferred.
CHOREO — reading vs editing
The Choreo tab has two modes, toggled by ✎ in the header. Reading = a programme: every figure, its steps, fixed info-slots, and the latest 2 comments inline. Editing = sections + figures with add / reorder. Sections are user-named free text (no Long/Short/Corner enum).
Scale mapping — px are roles, not specs. These frames are 320px lo-fi: 8px → text-2xs (12px) · 9–10px → xs · 11–12px → sm · 13px Caveat → note. Every tap target ≥ 44px (--bf-touch-target) — the whole cell / row is the hit area, never just the glyph. Chips = kind tint bg + kind border + dark ink (AA); solid colour + white text is banned.
Grid rules. Sub-beats (& · e · a) are nested — smaller count label, tinted row — never dimmed: grey means empty / disabled only. More than 5 columns? The grid pans horizontally; the count column stays sticky, the right edge fades. Type never shrinks below 2xs.

0 · Navigation — four tabs & where every screen connects
▦ CHOREO
Choreo list → tap a routine → Assemble (reading ⇄ editing via ✎). From Assemble: tap a figure → Figure detail → tap a step → Attribute editor. Tap any info-slot → Attribute info. In reading, tap a comment / "+N more" → Thread. List: ＋ → New-choreo sheet · ⋯ → Open / Fork sheet · share icon → Share.
▤ LIBRARY
Application-global figures grouped by figure-type family, filtered by dance. Each catalog figure has ↟ save → copies into My figures (toast "Saved to My figures · View"). My-figures cards show based-on lineage + used-in-N-choreos, with ✎ in-place edit (1.22).
▥ JOURNAL
Profile-coloured entries (lessons & practice). Tap an entry → Entry editor. "+ link" → Link picker: choose type (place / figure / attribute) → pick the target → pick scope. Links render as ↳ chips.
◓ PROFILE
Your name and profile colour — the colour every note and thread reply of yours is tinted with, across shared routines. Leader / Follower is a per-figure timeline toggle (remembered between sessions), not a profile setting.

System rules · part 2 — semantics that don't live on a screen
PERMISSIONS
Only editors & owners can change attributes; commenters and viewers are read-only. Edit rights flow from the routine to its figures — but being on a choreo never grants the right to delete a figure.
FIGURE COPIES & THE "CUSTOM" LABEL
An in-choreo copy is frozen — it stops inheriting from its origin. Editing a figure that lives outside this choreo makes that frozen, choreo-owned copy automatically; saving to My figures is a separate, deliberate action. A figure reads custom only once its technique or name drifts from the library pick (judged by meaning — type · timing · role · value) — distinct from variant.
DATA INTEGRITY
Soft-delete: removing an attribute tombstones it (so it can return and two editors never clobber each other) — it's just hidden. A custom figure stores only its diffs (changes / removals / additions / rename), not a full copy. Renamed or cleaned-up values still display correctly on old figures. Changing a value keeps a step's identity; moving its beat or role = remove + add.
TYPES — & WHAT ISN'T ONE
A type is single/multi, optional/required, and may be role-aware; it can be limited to certain dances (Rise & Fall doesn't apply to Tango). Custom types are choreo-scoped so partners see them. Entry/exit alignment and threaded notes are NOT attributes — they're separate things that live on the figure.
